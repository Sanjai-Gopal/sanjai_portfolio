Git & GitHub: Essential Commands Every Developer Should Know

February 15, 2026 7 min read Sanjai Gopal 980 views

Version control is non-negotiable in modern software development. Whether you're working solo or in a team, Git saves you from countless headaches. In this guide, I'll share the essential Git commands and GitHub workflows that every developer must know.

What is Version Control?

Version control is like a time machine for your code. It tracks every change, allows you to revert to previous versions, and enables multiple people to work on the same project without conflicts.

Key Concept: Git is the version control system (runs locally), while GitHub is a cloud platform for hosting Git repositories.

Installing Git

Download Git from git-scm.com. After installation, verify it works:

git --version

First-Time Configuration

Set your identity (this will be attached to your commits):

git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

# Optional: set default editor
git config --global core.editor "code --wait"  # For VS Code

# View all configurations
git config --list

Essential Git Commands Cheat Sheet

Starting a Project

CommandDescription
git initInitialize a new Git repository in current folder
git clone <url>Download an existing repository from GitHub

Making Changes

CommandDescription
git statusSee what files have changed
git add <file>Stage a file for commit
git add .Stage all changes
git commit -m "message"Commit staged changes with a message
git commit -am "message"Add and commit in one step (for tracked files)

Branching

CommandDescription
git branchList all branches (* indicates current)
git branch <name>Create a new branch
git checkout <branch>Switch to a branch
git checkout -b <name>Create and switch to new branch
git merge <branch>Merge branch into current branch
git branch -d <name>Delete a branch

Inspecting History

CommandDescription
git logView commit history
git log --onelineCompact commit history
git log --graphVisualize branch structure
git diffSee unstaged changes
git diff --stagedSee staged changes

Undoing Changes

CommandDescription
git restore <file>Discard changes in working directory
git restore --staged <file>Unstage a file
git reset HEAD~1Undo last commit (keep changes)
git reset --hard HEAD~1Undo last commit (discard changes)
git revert <commit-hash>Create new commit that undoes changes

Remote Repositories (GitHub)

CommandDescription
git remote add origin <url>Connect local repo to GitHub
git remote -vView remote connections
git push origin mainUpload commits to GitHub
git pull origin mainDownload latest from GitHub
git fetch originDownload but don't merge

Real-World Git Workflow

Here's how I use Git for my portfolio project:

# 1. Clone the repository
git clone https://github.com/Sanjai-Gopal/sanjai_portfolio.git
cd sanjai_portfolio

# 2. Create a branch for new feature
git checkout -b add-blog-section

# 3. Make changes and commit
git add .
git commit -m "Add blog section with initial posts"

# 4. Push branch to GitHub
git push origin add-blog-section

# 5. Create Pull Request on GitHub
# (Go to GitHub.com and create PR)

# 6. After review, merge and pull updates
git checkout main
git pull origin main

# 7. Delete the branch
git branch -d add-blog-section
Common Mistake: Forgetting to pull before starting new work. Always run git pull first!

Handling Merge Conflicts

Conflicts happen when two people change the same part of a file. Don't panic! Here's how to resolve:

# After git merge, you'll see:
# <<<<<<< HEAD
# Your changes
# =======
# Their changes
# >>>>>>> branch-name

# Edit the file to keep what you want
# Remove the conflict markers
# Then:

git add resolved-file.txt
git commit -m "Resolve merge conflict"

GitHub Features You Should Use

1. Pull Requests

The heart of collaboration. PRs allow code review before merging. Always:

  • Write clear descriptions
  • Link related issues
  • Request reviewers
  • Run CI checks

2. Issues

Track bugs and features like a pro. Use labels:

  • 🐛 bug
  • ✨ enhancement
  • 📚 documentation
  • 🎨 design

3. Actions (CI/CD)

Automate testing and deployment. Example workflow:

# .github/workflows/test.yml
name: Tests
on: [push]
jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2
      - name: Run tests
        run: |
          npm install
          npm test

4. GitHub Pages

Host websites for free (like this portfolio!). Just push to gh-pages branch.

Git Aliases for Speed

Create shortcuts for common commands:

git config --global alias.co checkout
git config --global alias.br branch
git config --global alias.ci commit
git config --global alias.st status
git config --global alias.unstage 'reset HEAD --'
git config --global alias.last 'log -1 HEAD'
git config --global alias.graph 'log --graph --oneline --all'

# Now use:
git co main     # instead of git checkout main
git st          # instead of git status

My Git Habits

  1. Commit Often: Small, focused commits are easier to understand and revert
  2. Write Good Messages: "Fix bug" is bad. "Fix login validation for empty email" is good
  3. Branch for Features: Never commit directly to main
  4. Pull Before Push: Avoid conflicts by syncing first
  5. Review Your Own PR: You'll catch mistakes before others do
Pro Tip: Use git stash to temporarily save changes without committing. Great for switching branches quickly!
